# راهنمای سئو رهنورد (فارسی)

## معرفی

این سند راهنمای کامل استراتژی سئو برای پروژه رهنورد است. هدف اصلی ما دو بخش است:

1. **کمک به افراد امدادجو**: اطمینان از دسترسی سریع به کمک اضطراری از طریق موتورهای جستجو
2. **جذب داوطلبان معتبر**: جلب افراد و سازمان‌های قابل اعتماد به شبکه ما

---

## استراتژی کلمات کلیدی

### کلمات کلیدی اصلی

#### برای افراد امدادجو

```
- کمک اضطراری ایران
- امداد فوری
- کمک اورژانسی
- سرپناه اضطراری
- کمک فوری پزشکی
- مشاوره حقوقی اضطراری
```

#### برای داوطلبان

```
- داوطلب شدن
- ثبت نام داوطلب
- احراز هویت داوطلب
- شبکه داوطلبین
- کمک به امدادجوان
```

### کلمات کلیدی محلی

برای هر شهر، این الگو را دنبال می‌کنیم:

```
- کمک اضطراری [نام شهر]
- امداد فوری [نام شهر]
- داوطلب در [نام شهر]
- سازمان کمک رسان [نام شهر]
```

## ساختار صفحات

### صفحات کلیدی ایجاد شده

1. **صفحه اصلی** (`/`)
   - معرفی کلی پلتفرم
   - دکمه SOS برجسته
   - دسته‌بندی خدمات

2. **دریافت کمک** (`/get-help`)
   - برای افراد امدادجو
   - توضیح فرآیند دریافت کمک
   - لیست خدمات اضطراری
   - نکات ایمنی

3. **ثبت نام داوطلب** (`/join-volunteer`)
   - برای داوطلبان
   - توضیح فرآیند احراز هویت
   - مزایای عضویت
   - داستان داوطلبان

4. **صفحات شهرها** (`/location/[city]`)
   - صفحه اختصاصی برای هر شهر
   - آمار محلی
   - خدمات موجود در آن شهر
   - راه‌های ارتباطی محلی

5. **بلاگ** (`/blog`)
   - مقالات آموزشی
   - راهنماهای امنیتی
   - نکات کمک‌های اولیه
   - مدیریت بحران

---

## بهینه‌سازی فنی

### متادیتا

تمام صفحات از سیستم متادیتای یکپارچه استفاده می‌کنند:

```typescript
export const metadata = generateMetadata({
  title: "عنوان صفحه",
  description: "توضیحات صفحه (۱۵۰-۱۶۰ کاراکتر)",
  keywords: ["کلمه کلیدی ۱", "کلمه کلیدی ۲"],
});
```

### داده‌های ساختار یافته (Schema.org)

برای هر نوع صفحه، اسکیمای مناسب اضافه شده:

- **صفحه اصلی**: Organization + WebSite Schema
- **صفحات شهر**: LocalBusiness Schema
- **بلاگ**: Article Schema
- **صفحه تماس**: FAQ Schema

### نقشه سایت (Sitemap)

نقشه سایت به صورت خودکار تولید می‌شود و شامل:

- تمام صفحات ثابت
- ۲۰+ صفحه شهر
- مقالات بلاگ

## بهینه‌سازی سرعت

### Core Web Vitals

برای کاربران با اینترنت ضعیف، بهینه‌سازی‌های زیر انجام شده:

1. **LCP (بارگذاری سریع)**
   - بهینه‌سازی تصاویر
   - بارگذاری تنبل (Lazy Loading)

2. **FID (پاسخ سریع)**
   - حداقل JavaScript
   - تقسیم کد

3. **CLS (پایداری بصری)**
   - ابعاد مشخص برای تصاویر
   - فضا برای محتوای پویا

## نکات مهم

### E-E-A-T (اعتماد و اعتبار)

برای سایت‌های YMYL (Your Money or Your Life) مانند رهنورد، این موارد حیاتی است:

1. **تجربه (Experience)**
   - داستان‌های واقعی داوطلبان
   - نظرات کاربران

2. **تخصص (Expertise)**
   - راهنماهای تخصصی
   - مدارک تیم

3. **اعتبار (Authoritativeness)**
   - همکاری با سازمان‌های معتبر
   - پوشش رسانه‌ای

4. **قابلیت اعتماد (Trustworthiness)**
   - فرآیند احراز هویت شفاف
   - اقدامات امنیتی
   - اطلاعات تماس واضح
